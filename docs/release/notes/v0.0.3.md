# v0.0.3

- documents the `ccc-config.toml` edit path with a copy-paste `ccc setup`, Codex CLI restart, and `ccc check-install` refresh flow
- strengthens generated raider instructions around module boundaries, helper extraction, avoiding giant functions, and avoiding unrelated rewrites
- tightens generated specialist prompts so agents keep context compact and preserve the operator's language for user-facing text
- extends routing signals for Korean, English, Japanese, and Chinese requests across read, review, planning, mutation, docs, filesystem, and git/gh operations
- expands default config with companion reader/operator roles and git/gh-oriented companion routing using `gpt-5.4-mini` / `medium`
- treats `gh` as a first-class routed tool signal, including read vs mutation ownership split (`companion_reader` / `companion_operator`) and default `gh` routing entries
- backfills missing `companion_agents` and `tool_routing` defaults in existing `ccc-config.toml` files without overwriting user-defined values
- makes `ccc status --text` print an explicit token-unavailable reason even when no delegated-worker usage events exist
- extends quiet CLI lifecycle lines (`status/start/orchestrate/subagent-update`) with compact token usage or explicit unavailable reason codes
- updates generated specialist/task prompts to handle user-provided external paths safely: search/read/copy when sandbox-allowed, otherwise report exact blocked path and approval requirement
- adds multilingual README coverage and role model/reasoning tables for English, Korean, and Japanese
- hardens release packaging with a minimal release surface, stripped staged binary, and pre-release sensitive-string scan script
